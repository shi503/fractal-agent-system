# SUBAGENT.md - The Scoped Executor

This document is the context file for a Sub-Agent. It is generated by a Feature Lead and contains a single, well-defined task to be executed. The Sub-Agent reads this file, performs the task, and then terminates.

---

## 1. Task Specification

*A clear and unambiguous description of the task to be performed. This should be a single, atomic unit of work.*

**Example:**

> Implement the `hashPassword` function in `/src/utils/auth.ts`. The function should take a plain-text password as input and return a hashed password using the bcrypt library. The salt rounds should be 10.

## 2. File Manifest

*A list of the files that the Sub-Agent is permitted to read from and write to.*

**Example:**
- **Read/Write:** `/src/utils/auth.ts`
- **Read-Only:** `/package.json` (to check for dependencies)

## 3. Acceptance Criteria

*The specific criteria that must be met for the task to be considered complete. This should be a list of verifiable outcomes.*

**Example:**
- [ ] The `hashPassword` function is implemented in `/src/utils/auth.ts`.
- [ ] The function uses the `bcrypt` library.
- [ ] The salt rounds are set to 10.
- [ ] The function is exported from the module.
